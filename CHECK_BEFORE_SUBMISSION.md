# Final Submission Checklist

Before uploading to GitHub/LMS, complete these final manual items:

- [ ] Update team details in `README.md`.
- [ ] Create one public GitHub repository.
- [ ] Push folders: `/frontend`, `/backend`, `/database`, `/report`, `/demo`.
- [ ] Ensure at least 10 meaningful commits.
- [ ] Ensure every member has their own commits.
- [ ] Capture real UI screenshots and place them in `report/screenshots/`.
- [ ] Add screenshot images to the final report PDF if your teacher requires screenshots inside the PDF itself.
- [ ] Record 5-10 minute video demo.
- [ ] Paste final video link into `README.md` and `demo/VIDEO_LINK.txt`.
- [ ] Add live deployed link if you deploy the app.
- [ ] Test backend and frontend on the supervisor's laptop before submission.

Important: `node_modules`, `.angular`, `dist`, `bin`, and `obj` should not be committed or submitted in the ZIP.
